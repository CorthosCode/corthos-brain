
## Зачем нужны тесты?

Тестирование — это способ убедиться, что твой код работает так, как ожидается.

Основные цели:

* Поймать ошибки на раннем этапе.
* Поддерживать стабильность кода при изменениях (рефакторинг, новые фичи).
* Повысить доверие к приложению.


***

## Виды тестирования (общее представление)

* **Unit-тесты** — тестируют отдельные классы/методы изолированно.
* **Integration-тесты** — проверяют работу нескольких компонентов вместе.
* **End-to-End (E2E)** — тестируют весь поток (например, REST API + база данных).
* **Smoke/Regression** — быстрые проверки после изменений.
