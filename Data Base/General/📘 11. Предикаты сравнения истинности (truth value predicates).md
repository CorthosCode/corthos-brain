
Обычно мы пишем условия вида:

```sql
WHERE age > 30
```

и ожидаем, что это условие даст нам строки.

Но из-за **трёхзначной логики (TRUE / FALSE / UNKNOWN)** в SQL бывают ситуации, когда результат выражения не `TRUE` и не `FALSE`, а `UNKNOWN`.

Чтобы можно было явно проверять «истинность» условия, в SQL есть специальные **предикаты сравнения истинности**.


***

## Предикаты сравнения истинности в PostgreSQL

В PostgreSQL есть 3 оператора:


1. **`IS TRUE`**
   Проверяет, является ли выражение истинным (`TRUE`).

```sql
SELECT (1 = 1) IS TRUE;     -- TRUE
SELECT (1 = 2) IS TRUE;     -- FALSE
SELECT (NULL = 1) IS TRUE;  -- FALSE (ведь там UNKNOWN)
```



1. **`IS FALSE`**
   Проверяет, является ли выражение ложным (`FALSE`).

```sql
SELECT (1 = 2) IS FALSE;    -- TRUE
SELECT (1 = 1) IS FALSE;    -- FALSE
SELECT (NULL = 1) IS FALSE; -- FALSE
```



1. **`IS UNKNOWN`**
   Проверяет, является ли результат условия `UNKNOWN`.

```sql
SELECT (NULL = 5) IS UNKNOWN; -- TRUE
SELECT (1 = 2) IS UNKNOWN;    -- FALSE
SELECT (1 = 1) IS UNKNOWN;    -- FALSE
```


***

## Важные детали

* Эти предикаты **не путаем** с `IS NULL`.
  * `IS NULL` проверяет *значение столбца*
  * `IS UNKNOWN` проверяет *результат логического выражения*
* Они полезны, когда вы хотите явно обработать случаи с `NULL`.


***

## Пример использования в запросах

```sql
SELECT name, age
FROM users
WHERE (age > 30) IS TRUE;      -- выберет только тех, у кого age > 30
```

```sql
SELECT name
FROM users
WHERE (age > 30) IS UNKNOWN;   -- выберет строки, где age IS NULL
```

```sql
SELECT name
FROM users
WHERE (age > 30) IS FALSE;     -- выберет строки, где age <= 30
```
