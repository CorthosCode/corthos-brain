
**Модель данных** — это концептуальное (абстрактное) представление данных, их структуры, взаимосвязей и ограничений.

Она определяет:
* какие данные можно хранить;
* как они связаны;
* какие операции допустимы.


***

## Основные уровни моделей данных


### 1. Концептуальная модель (логическая, предметная)

* Показывает **что хранится**, а не как.
* Используется для общения с бизнес-аналитиками и заказчиками.
* Пример: ER-диаграмма (сущность → атрибуты → связи).
* Пример:
  * Сущность: "Клиент" (ФИО, телефон).
  * Сущность: "Заказ" (номер, дата, сумма).
  * Связь: "Клиент оформляет Заказ".


### 2. Логическая модель

* Ближе к реализации, но ещё не конкретная.
* Зависит от выбранной парадигмы БД (реляционная, документо-ориентированная, графовая).
* В реляционной СУБД: таблицы, связи, первичные и внешние ключи.
* В документо-ориентированной: коллекции и документы (JSON-подобные).


### 3. Физическая модель

* То, как данные реально хранятся.
* Включает: индексы, типы данных, разбиение таблиц, партиции, файловую структуру, механизмы доступа.
* Обычно зависит от конкретной СУБД (PostgreSQL, Oracle, MongoDB, Neo4j и т. д.).


***

## Классы моделей данных

Существует несколько **основных моделей**, которые легли в основу разных поколений БД:


### 1. Иерархическая модель

* Данные организованы в **дерево**.
* Каждый родитель может иметь много детей, но у ребёнка только один родитель.
* Пример: IBM IMS (1968).
* Недостаток: сложно моделировать "многие-ко-многим".


### 2. Сетевая модель

* Данные как **граф**: сущности и связи могут иметь произвольную структуру.
* Использовалась в IDMS, TurboIMAGE.
* Гибче иерархической, но сложная для программистов.


### 3. Реляционная модель (основная сегодня)

* Основана на **таблицах (отношениях)**.
* Термины: строки = кортежи, столбцы = атрибуты.
* Основные элементы:
  * ключи (PK, FK, уникальные);
  * ограничения (NOT NULL, CHECK);
  * нормализация.
* Примеры СУБД: MySQL, PostgreSQL, Oracle, SQL Server.


### 4. Объектно-ориентированная модель

* Появилась для интеграции с ООП-языками.
* Поддержка: классы, наследование, методы, инкапсуляция.
* Примеры: ObjectDB, db4o.


### 5. Документо-ориентированная модель

* Данные = документы (JSON, BSON, XML).
* Хорошо подходит для хранения полуструктурированных данных.
* Примеры: MongoDB, CouchDB.


### 6. Ключ-значение (Key-Value)

* Самая простая модель.
* Данные хранятся как пары {ключ → значение}.
* Пример: Redis, Riak.


### 7. Графовая модель

* Данные представлены узлами и рёбрами.
* Удобна для социальных сетей, рекомендаций, маршрутизации.
* Примеры: Neo4j, ArangoDB.


### 8. Колонно-ориентированная модель

* Хранение по колонкам, а не по строкам.
* Удобно для аналитики (OLAP).
* Примеры: Cassandra, HBase, ClickHouse.


***

## Классификация по применению

* **OLTP** (Online Transaction Processing) — транзакционные системы (реляционные БД).
* **OLAP** (Online Analytical Processing) — аналитические системы (колоночные БД, хранилища).
* **HTAP** (Hybrid Transactional/Analytical Processing) — гибридные (TiDB, SAP HANA).


***

## Взаимосвязь

**Концептуальная модель (ER-диаграмма)** → **Логическая модель (таблицы, связи)** → **Физическая модель (типы данных, индексы, партиции)**.
